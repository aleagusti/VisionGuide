# Contributing to VisionGuideThank you for your interest in contributing to VisionGuide!  This document explains how to submit improvements, report issues, andcollaborate with the project in a safe and structured way.VisionGuide welcomes contributions from the community, with the goal ofadvancing accessibility and assistive technologies for visual navigation.---## 1. Code of ConductBy participating in this project, you agree to:- Be respectful, constructive, and collaborative  - Avoid harassment, discrimination, and abusive behavior  - Focus on technical discussion and problem solving  - Respect differing perspectives  If any issues arise, please report them to the project maintainers.---## 2. How to Contribute### 2.1 Reporting BugsPlease open an issue including:- A clear description of the problem  - Steps to reproduce  - Expected vs. actual behavior  - Screenshots or logs (if relevant)  - System information (OS, Python version, hardware)  This helps maintainers diagnose the issue efficiently.---### 2.2 Suggesting FeaturesFeature requests are welcome!  Include:- What problem the feature solves  - Why it benefits VisionGuide  - Potential implementation ideas (optional)  - Any references or examples of similar solutions  ---### 2.3 Submitting Code (Pull Requests)Before submitting a PR:1. Ensure your changes build and run without errors  2. Follow the coding style used in the project  3. Write clear commit messages  4. Reference related issues (e.g., "Fixes #12")  5. Test your changes if applicable  6. Ensure your contribution respects all licenses listed in `NOTICE`  Pull Requests must be small, focused, and readable.---## 3. Legal Requirements### 3.1 License AgreementBy submitting code, documentation, or other content to this repository,**you agree that your contribution is licensed under the project's primaryopen-source license:****AGPL-3.0-or-later**This ensures:- Your code remains open-source and compatible with the project  - No contributor can later revoke permission or demand removal  - The project can continue to be dual-licensed legally  To avoid legal conflicts, **do not submit any code that you do not havepermission to contribute**.---### 3.2 Corporate ContributorsIf you contribute as part of your employment or on behalf of a company:- Confirm you have permission from your employer  - Ensure no proprietary or confidential information is included  If needed, the project may request a simple **Corporate Contributor Declaration**.---## 4. Code Style GuidelinesTo maintain consistency:- Use descriptive variable names  - Follow PEP8 formatting for Python  - Write docstrings for public functions and classes  - Structure imports clearly  - Avoid unnecessarily large dependencies  - Keep code modular and readable  ---## 5. Branching Model- `main` -> stable releases  - `dev` -> active development  - Feature branches -> `feature/<short-description>`  - Bugfix branches -> `bugfix/<short-description>`  All pull requests should target the `dev` branch unless specified otherwise.---## 6. Security IssuesIf you discover a **security vulnerability**, do NOT open a public issue.Instead, report it privately to:**agustialejandro@gmail.com**Please include:- Description of the vulnerability  - Steps to reproduce  - Potential impact  - Suggested fix (optional)  ---## 7. Documentation ContributionsDocumentation improvements are extremely valuable.You can contribute:- Clarifications  - Tutorials  - Architecture explanations  - Diagrams (SVG/PNG)  - Examples or demos  All documentation is licensed under **CC BY-NC 4.0**, unless stated otherwise.---## 8. Thank YouYour contributions help make VisionGuide a better and more accessibleproject. We deeply appreciate your support and collaboration.If you have any questions, feel free to open an issue or reach out privately.
